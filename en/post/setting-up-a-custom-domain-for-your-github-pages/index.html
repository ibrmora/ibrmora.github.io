Setting up a custom domain for your GitHub Pages

22/05/2018

GitHub provides a free static website hosting and GitHub subdomain services. GitHub can get an address for your website with a prefix corresponding to your username if the repository name of your site files has a prefix corresponding to your username and a similar username.github.io.

For example,
username           ibrmora
repository name           ibrmora.github.io
GitHub subdomain           https://ibrmora.github.io

If you want to register a custom domain name for your website, you’ll need to choose your custom domain and registering it with a DNS provider. After that you will have many options for your website address, most importantly:

  • An apex domain   (https://ibrmo.com)

In this case you will not be able to use the secure connection directly from GitHub, but you need to use an intermediary service such as CloudFlare to be able to add a secure connection certificate there.

  • A www subdomain   (https://www.ibrmo.com)

You will be able to use secure connection from GitHub here. The URL settings generally recommend using the www prefix and you can see this in large sites like Google.

Then add a file named CNAME that contains your custom domain (www.ibrmo.com).

You now need to configure your domain with your DNS provider as follows:

Record Type Host Value TTL
A @ 192.30.252.153 Automatic
A @ 192.30.252.154 Automatic
CNAME www ibrmora.github.io. Automatic

It is important that you do not adjust the settings before you add the custom domain to the CNAME file.

To activate the secure connection go to the GitHub Pages section in the repository settings and put a check mark in the «Enable HTTPS» checkbox. If you choose the apex domain, this option will not be available.

In both options above, GitHub will redirect one of the domains to the other.

For example,

http://ibromo.com   It will be redirected to   https://www.ibrmo.com.
