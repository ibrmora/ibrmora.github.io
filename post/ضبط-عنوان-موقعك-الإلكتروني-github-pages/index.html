ضبط عنوان موقعك الإلكتروني في GitHub Pages

11/05/2018

يوفر GitHub خدمة استضافة و عنونة المواقع الثابتة مجانا.حيث يُمَكِّنْكَ GitHub من الحصول على عنوان لموقعك الإلكتروني ببادئة مطابقة لاسم المستخدم الخاص بك في حال كان اسم المستودع الحاوي لملفات موقعك ببادئة مطابقة لاسم المستخدم و صيغة مماثلة لـ  username.github.io .

مثال ذلك :
اسم المستخدم ibrmora
اسم المستودع           ibrmora.github.io
عنوان الموقع           https://ibrmora.github.io

عند رغبتك بتسجيل عنوان خاص لموقعك يلزمك القيام بتسجيل عنوانك لدى أحد مزودي خدمة تسجيل العناوين. بعد ذلك سيكون لديك العديد من الخيارات لعنوان موقعك أهمها:

  • استخدام العنوان الرئيسي   (https://ibrmo.com)

في هذه الحالة لن تتمكن من استخدام الإتصال الآمن بشكل مباشر من GitHub بل يلزمك استخدام خدمة وسيطة كـ CloudFlare لتتمكن من إضافة شهادة الإتصال الآمن هناك.

  • استخدام العنوان الفرعي   (https://www.ibrmo.com)

ستتمكن هنا من استخدام الإتصال الآمن من GitHub كما أن إعدادات عنوانين المواقع الإلكترونية بشكل عام توصي باستخدام بادئة www و يمكنك ملاحظة ذلك في المواقع الكبيرة كغوغل.

بعد ذلك قم بإضافة ملف باسم CNAME يحتوي على عنوانك (www.ibrmo.com) .

الآن يلزمك ضبط إعدادات مزود خدمة تسجيل العناوين على النحو التالي :

TTL Value Host Record Type
Automatic 192.30.252.153 @ A
Automatic 192.30.252.154 @ A
Automatic ibrmora.github.io.‎ www CNAME

من المهم ألا تقوم بضبط الإعدادات قبل إضافة العنوان إلى الملف CNAME .

لتفعيل الإتصال الآمن قم بالذهاب إلى القسم الخاص بـ GitHub Pages في إعدادات المستودع وضع علامة صح في خانة تأشير ‎.Enforce HTTPS في حال قمت باختيار العنوان الرئيسي لن يكون هذا الخيار متاحاً.

في كلا الخيارين أعلاه سيقوم GitHub بإعادة توجيه أحد العنوانين إلى الآخر.

مثال ذلك :

http://ibrmo.com   سيتم إعادة التوجيه إلى   https://www.ibrmo.com .
